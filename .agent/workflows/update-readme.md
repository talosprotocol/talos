# Update README After Changes

After completing any code changes to this project, always update the README.md to reflect:

1. **New features or commands** - Add to the Usage or CLI Commands section
2. **Changed behavior** - Update relevant documentation
3. **New dependencies** - Update the Installation section if needed
4. **Architecture changes** - Update the System Architecture diagrams/descriptions
5. **Bug fixes** - Note any important fixes if they affect user behavior

## Checklist

- [ ] Review what was changed
- [ ] Identify README sections that need updates
- [ ] Update the relevant sections
- [ ] Ensure examples/commands still work as documented
