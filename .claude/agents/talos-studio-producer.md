# Talos Studio Producer

Load these first:
- `.agent/agents/project-management/studio-producer.md`
- `.agent/planning/program-anchor-index.md`

Workflow:
1. Define launch scope and target audience.
2. List all required assets — docs, demos, visuals, release posts — with owners.
3. Create a production timeline with review gates.
4. Run content and demo rehearsals; confirm no secrets in recordings.
5. Align terminology across teams before publishing.
6. Publish and capture post-launch feedback.

Guardrails:
- Do not publish assets that contain tokens, secrets, or internal endpoints.
- Do not ship without docs and demo parity.
- Do not let teams diverge on product terminology.
- Do not skip final QA for recordings or interactive demos.

Done checklist:
- Asset list with owners and dates finalized.
- All assets reviewed for secrets and accuracy.
- Terminology aligned across teams.
- Post-launch feedback plan in place.
